| 사용자 의도 | 실행할 동작 |
|------------|-----------|

## 전체 현황 조회

### 정보 수집 (모든 항목을 실제 파일에서 읽는다. 추측 금지)

| 항목 | 소스 파일 | 추출 방법 |
|------|----------|----------|
| 모델 목록 | `/root/.openclaw/openclaw.json` | `models.providers` 각 키의 id, 모델 목록 |
| 기본 모델 | `/root/.openclaw/openclaw.json` | `agents.defaults.model.primary` |
| 모델 배치 | `AGENTS.md` | `## 모델 배치` 섹션 내용 |
| 스킬 목록 | `skills/*/SKILL.md` | 각 폴더의 SKILL.md frontmatter에서 name, description 추출 |
| 활성/비활성 | `skills/*/SKILL.md` | frontmatter `disable-model-invocation: true` 여부 |
| 트리거 정보 | `skills/*/SKILL.md` | 본문에서 트리거 조건 추출 (cron, 주기, 요청 시) |
| 기억 항목 수 | `MEMORY.md` | `## 기억 항목` 섹션의 항목 수 카운트 |
| 사용자 규칙 | `AGENTS.md` | `## 사용자 규칙` 섹션 내용 |

### 표시 형식

```
시스템 스킬: _template 폴더와 아래 9개는 "시스템 스킬"로 분류
  onboarding, profile-editor, rule-editor, skill-creator,
  skill-editor, skill-toggle, memory-manager, agent-router, status-dashboard

그 외: "사용자 스킬"로 분류

표시:
  "[현재 상태]"
  ""
  "모델:"
  "- [모델명] ([provider]) -- 기본"
  "- [추가 모델이 있으면 나열]"
  ""
  "모델 배치:"
  (AGENTS.md에 배치 규칙이 있으면 각 규칙 표시)
  (없으면 "- 모든 작업 → [기본 모델]")
  ""
  "사용자 스킬 ([N]개 활성 / [M]개 비활성):"
  (활성)
  "  - [스킬명]: [트리거 요약] -- [description 한 줄]"
  (비활성)
  "  - [스킬명]: 비활성화됨"
  (없으면 "  없음")
  ""
  "시스템 스킬: [N]개 (프로필 편집, 규칙 편집, 스킬 관리 등)"
  ""
  "사용자 규칙: [N]개"
  (있으면 각 규칙 한 줄씩)
  (없으면 "없음")
  ""
  "기억 항목: [N]건"
  ""
  "자세히 보고 싶은 스킬이 있으면 말해주세요."
```

## 스킬 세부 정보 조회

### 정보 수집

```
대상 스킬의 SKILL.md를 읽고 아래 항목을 추출한다:

| 항목 | 추출 위치 |
|------|----------|
| 이름 (name) | frontmatter |
| 설명 (description) | frontmatter |
| 활성 상태 | frontmatter disable-model-invocation |
| 필요 환경변수 | frontmatter metadata.openclaw.requires.env |
| 필요 바이너리 | frontmatter metadata.openclaw.requires.bins |
| 트리거 조건 | 본문에서 추출 |
| 입력 정보 | 본문에서 추출 |
| 출력 형식 | 본문에서 추출 |
| 보충 파일 | 스킬 폴더 내 SKILL.md 외 파일 목록 |

환경변수 등록 여부 확인:
  /root/.openclaw/.env 또는 환경변수에서 해당 키가 존재하는지 확인
  → "(등록됨)" 또는 "(미등록)"
```

### 표시 형식

```
"[스킬명] 상세:"
""
"| 항목 | 값 |"
"|------|-----|"
"| 이름 | [name] |"
"| 설명 | [description] |"
"| 트리거 | [조건] |"
"| 입력 | [정보] |"
"| 출력 | [형식] |"
"| 환경변수 | [변수명] (등록됨/미등록) |"
"| 필요 패키지 | [바이너리 목록] |"
"| 상태 | 활성/비활성 |"
"| 위치 | skills/[스킬명]/ |"
"| 보충 파일 | [파일 목록] (없으면 '없음') |"
""
"수정하려면 '[스킬명] 수정해줘'라고 말해주세요."
```

## 스킬 역할 일괄 설명

> 트리거: "스킬 역할 설명해줘", "각 스킬이 뭐하는 거야?", "스킬 목록별 역할"

### 정보 수집

```
읽기 대상:
  - skills/*/SKILL.md → 각 스킬의 name, description, 트리거 조건
  - AGENTS.md "## 모델 배치" → 각 스킬의 할당 모델
  - 할당 모델 미지정 스킬 → "기본 ([기본 모델명])"
```

### 표시 형식

```
사용자 스킬과 시스템 스킬을 나누어 표시한다.

"[사용자 스킬 역할 안내]"
""
"| 스킬 | 역할 | 트리거 | 할당 모델 |"
"|------|------|--------|----------|"
"| weather-alert | 매일 아침 서울 날씨 조회 후 알림 | 매일 07:00 | Claude Sonnet |"
"| email-summary | 읽지 않은 이메일 요약 전송 | 매시간 | 기본 (qwen3.5:9b) |"
(없으면 "사용자 스킬이 없습니다. 추가하려면 '기능 추가해줘'라고 말해주세요.")
""
"[시스템 스킬 역할 안내]"
""
"| 스킬 | 역할 |"
"|------|------|"
"| onboarding | 첫 대화 시 프로필 수집 + 기능 안내 |"
"| profile-editor | 호칭/톤/프로필 변경 |"
"| rule-editor | 행동 규칙, 알림 규칙 추가/수정/삭제 |"
"| skill-creator | 새 스킬 생성 (정형화된 질문 플로우) |"
"| skill-editor | 기존 스킬 설정/로직 수정 |"
"| skill-toggle | 스킬 활성화/비활성화 |"
"| memory-manager | 기억 저장/조회/삭제 |"
"| agent-router | 모델 목록 확인 + 작업별 모델 배치 |"
"| status-dashboard | 전체 현황 조회 + 세부 정보 |"
""
"특정 스킬을 자세히 보려면 스킬 이름을 말해주세요."
```

## 모델별 스킬 할당 현황

> 트리거: "모델별로 보여줘", "에이전트 목록", "모델이 뭐 하고 있어?", "에이전트별 할당"

### 정보 수집

```
읽기 대상:
  1. /root/.openclaw/openclaw.json → models.providers → 등록된 모델 전체 목록
  2. AGENTS.md → "## 모델 배치" 섹션 → 스킬별 할당 규칙
  3. agents.defaults.model.primary → 기본 모델 확인

정리 방법:
  - 각 모델에 할당된 스킬 목록을 그룹핑
  - 명시적 할당이 없는 스킬 → "기본 모델" 그룹에 포함
  - 시스템 스킬은 별도 표시 (모델 배치 대상 아님)
```

### 표시 형식

```
"[모델별 스킬 할당 현황]"
""
"[모델명] ([provider]) -- 기본:"
"  - 일상 대화 (기본)"
"  - weather-alert (날씨 알림)"
"  - email-summary (이메일 요약)"
""
"[모델명] ([provider]):"
"  - document-writer (문서 작성)"
"  - skill-creator 작업 위임 (스킬 생성 시)"
""
"시스템 스킬 (9개): 모델 배치 없이 기본 모델로 실행"
""
"배치를 변경하려면 '스킬명 모델 바꿔줘'라고 말해주세요."
"모델을 추가하려면 .env에 API 키를 등록해주세요."
```

## 제약

```
MUST:
  - 모든 정보를 실제 파일에서 읽어서 표시 (추측/캐싱 금지)
  - 시스템 스킬과 사용자 스킬을 구분하여 표시
  - 환경변수는 등록 여부만 표시 (값은 절대 표시 안 함)
  - 세부 정보에서 수정으로 연계할 수 있도록 안내 문구 포함

NEVER:
  - API 키 값 표시
  - 파일에서 읽지 않은 정보를 추측으로 표시
  - 존재하지 않는 스킬 정보 표시
```
