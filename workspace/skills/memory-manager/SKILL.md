# 기억 관리

> 트리거: "기억해둬", "메모해둬", "뭐 기억해?", "기억 보여줘", "잊어줘", "기억 삭제"

---

## 동작 분기

| 사용자 의도 | 실행할 동작 |
|------------|-----------|
| 저장 ("기억해둬", "메모해둬") | 저장 플로우 |
| 조회 ("뭐 기억해?", "기억 보여줘", "~에 대해 뭐 알아?") | 조회 플로우 |
| 삭제 ("잊어줘", "지워줘", "기억 삭제") | 삭제 플로우 |

---

## 저장 플로우

### Step 1: 저장할 내용의 핵심 파악

사용자 메시지에서 기억할 핵심 내용을 추출한다.

```
예시:
  "다음 주 수요일에 클라이언트 미팅 있어. 기억해둬"
  → 핵심: "4/9(수) 클라이언트 미팅"
```

### Step 2: 파일명 결정

```
파일명 규칙:
  - 특정 주제가 있고, 해당 주제의 기존 파일이 있음 → 기존 파일에 추가
    예: 프로필 관련 → memory/user-profile.md (이미 존재하면 추가)
  - 특정 주제가 있고, 기존 파일 없음 → 새 파일 생성
    예: 프로젝트 A 관련 → memory/project-a.md
  - 일회성 메모 → 오늘 일별 노트에 추가
    예: memory/2026-04-02.md
```

### Step 3: memory/ 파일에 상세 기록

```
기록 형식:
  ## YYYY-MM-DD HH:MM 추가
  - 내용: (기억할 상세 내용)
  - 맥락: (대화에서 언급된 배경 정보가 있으면 포함)
```

### Step 4: MEMORY.md 인덱스 추가

```
형식: "- [요약 한 줄] → memory/파일명.md"
예시: "- 4/9(수) 클라이언트 미팅 → memory/2026-04-02.md"

기존에 같은 파일을 참조하는 항목이 있으면 → 요약 업데이트 (중복 항목 생성 금지)
```

### Step 5: 완료 알림

```
형식: "기억했습니다. '[요약 내용]' → memory/파일명.md"
```

---

## 조회 플로우

### Step 1: 조회 범위 파악

```
전체 조회 ("뭐 기억해?", "기억 목록")
  → MEMORY.md 인덱스 전체 표시

특정 주제 조회 ("~에 대해 뭐 알아?", "지난주에 뭐 말했지?")
  → MEMORY.md에서 관련 항목 검색
  → 매칭되는 memory/*.md 파일 읽어서 상세 내용 표시
```

### Step 2: 결과 표시

```
전체 조회 형식:
  "현재 기억 항목 [N]건:"
  "- [요약] (memory/파일명.md)"
  "- [요약] (memory/파일명.md)"
  "..."
  ""
  "자세한 내용을 보려면 항목을 말해주세요."

특정 주제 형식:
  "[주제] 관련 기억:"
  (memory/*.md에서 읽은 상세 내용)
```

## 삭제 플로우

### Step 1: 삭제 대상 확인

사용자 요청에서 삭제 대상을 파악한다.

```
명확한 경우: "클라이언트 미팅 기억 지워줘" → 해당 항목 특정
불명확한 경우: MEMORY.md 인덱스 보여주고 → "어떤 항목을 삭제할까요?" 선택 요청
```

### Step 2: 삭제 확인

```
"이 기억을 삭제할까요?"
"- [삭제 대상 내용 요약]"
""
"삭제하면 복구할 수 없습니다."
```

### Step 3: 승인 → 삭제 실행

```
삭제 순서:
  1. memory/*.md에서 해당 내용 삭제 (파일 내 일부면 해당 부분만, 파일 전체면 파일 삭제)
  2. MEMORY.md 인덱스에서 해당 항목 제거
```

### Step 4: 완료 알림

```
"삭제 완료."
```

---

## MEMORY.md 용량 관리

```
조건: MEMORY.md 내용이 15,000자를 초과할 때 (20,000자 제한의 75%)

동작:
  1. 가장 오래된 항목부터 memory/archive.md로 이동
  2. MEMORY.md에서 해당 항목 제거하고 대신 "- 과거 기억 아카이브 → memory/archive.md" 한 줄 유지
  3. 사용자에게 "기억 인덱스 정리 완료. 오래된 항목은 memory/archive.md에 보관됩니다." 안내
```

---

## 제약

```
MUST:
  - MEMORY.md는 인덱스만 유지 (상세 내용은 반드시 memory/ 파일에)
  - 삭제 시 반드시 사용자 확인 (자동 삭제 금지)
  - 같은 파일을 참조하는 중복 인덱스 항목 생성 금지
  - MEMORY.md 20,000자 이내 유지

NEVER:
  - 민감 정보 (API 키, 비밀번호, 토큰) 기억
  - 사용자 확인 없이 기억 삭제
  - memory/ 파일 내용을 추측으로 작성 (실제 대화 내용만 기록)
```
