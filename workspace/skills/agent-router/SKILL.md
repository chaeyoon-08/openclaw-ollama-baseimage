# 에이전트/모델 배치

> 트리거: "Claude한테 시켜", "모델 배치", "어떤 모델 있어?", "모델 바꿔", "모델 목록"

---

## 동작 분기

| 사용자 의도 | 실행할 동작 |
|------------|-----------|
| 모델 목록/현재 배치 확인 | 상태 조회 |
| 특정 작업의 모델 변경 | 배치 변경 |
| 새 모델 등록 직후 | 추천 배치 제안 |

## 상태 조회

### 정보 수집

```
읽기 대상:
  1. /root/.openclaw/openclaw.json → models.providers 키
    - 각 provider의 id, 모델 목록 추출
    - Ollama 모델: models.providers.ollama.models[].id
    - 외부 모델: models.providers.[provider].models (빈 배열이면 "자동 감지" 표시)

  2. AGENTS.md → "## 모델 배치" 섹션
    - 현재 작업별 배치 규칙 읽기
    - 섹션 비어있거나 "(없음)" → 모든 작업이 기본 모델
```

### 모델 1개 (Ollama만)

```
메시지:
  "등록된 모델: [모델명] (Ollama)"
  ""
  "현재 모든 작업이 이 모델로 처리됩니다."
  ""
  "추가 모델을 사용하려면 .env에 API 키를 등록해주세요."
  "  경로: /root/.openclaw/.env"
  "  등록 가능: Anthropic, OpenAI, Google, Mistral, DeepSeek, Groq"
  "  등록 후: bash /usr/local/bin/reload.sh"
```

### 모델 여러 개

```
메시지:
  "등록된 모델:"
  "- [모델명] ([provider]) -- 기본"
  "- [모델명] ([provider])"
  ""
  "스킬별 모델 배치:"
  (AGENTS.md "## 모델 배치"에 규칙이 있으면)
  "- [스킬명] → [모델명]"
  "- [스킬명] → [모델명]"
  "- 기타 스킬 → [기본 모델명]"
  (없으면)
  "- 모든 스킬 → [기본 모델명] (배치 규칙 미설정)"
  ""
  "배치를 변경하려면 '[스킬명]은 [모델명]으로' 라고 말해주세요."
  "모델별 현황은 '모델별로 보여줘'라고 해주세요."
```

## 배치 변경

### Step 1: 변경 요청 파악

```
파악할 항목:
  - 작업 종류: 어떤 유형의 작업인지 (일상 대화, 스킬 생성, 이메일 작성 등)
  - 배치할 모델: 어떤 모델로 배치할지

□ 지정된 모델이 등록되어 있는가?
  → Yes: Step 2
  → No: "해당 모델이 등록되어 있지 않습니다. 먼저 .env에 API 키를 등록해주세요." (종료)
```

### Step 2: 변경 사항 표시 + 승인

```
"배치를 변경합니다."
""
"| 작업 | 현재 | 변경 후 |"
"|------|------|---------|"
"| [작업] | [현재 모델] | [새 모델] |"
""
"적용할까요?"
```

### Step 3: AGENTS.md 수정

```
수정 위치: AGENTS.md → "## 모델 배치" 섹션

형식:
  ## 모델 배치
  - weather-alert → anthropic/claude-sonnet-4-20250514
  - email-summary → anthropic/claude-sonnet-4-20250514
  - 기본(미지정 스킬) → ollama/qwen3.5:9b

기존 "(없음)" 텍스트가 있으면 제거하고 배치 규칙 추가
이미 같은 스킬에 대한 규칙이 있으면 덮어쓰기
```

### Step 4: 변경 후 전체 배치 표시

```
"적용 완료."
""
"[현재 모델 배치]"
"- [작업] → [모델]"
"- [작업] → [모델]"
"- 기타 → [기본 모델]"
```

### Step 5: 기록

```
memory/YYYY-MM-DD.md에 배치 변경 이력 추가
MEMORY.md 인덱스 업데이트
```

## 추천 배치 제안 (신규 API 모델 등록 시)

```
조건: 이전에 Ollama만 있었는데 새 API 모델이 등록됨
  (사용자가 "모델 추가했어" 또는 reload.sh 실행 후 새 모델 감지)

메시지:
  "새 모델이 등록되었습니다: [모델명] ([provider])"
  ""
  "현재 사용자 스킬에 대한 추천 배치:"
  (사용자 스킬이 있는 경우)
  "| 스킬 | 현재 모델 | 추천 모델 | 이유 |"
  "|------|----------|----------|------|"
  "| [스킬명] | Ollama | [API 모델] | [이유] |"
  (없는 경우)
  "아직 사용자 스킬이 없습니다. 스킬을 추가하면 배치를 설정할 수 있습니다."
  ""
  "기본 모델(미지정 스킬용)은 Ollama를 유지하는 것을 추천합니다. (빠르고 비용 없음)"
  ""
  "이대로 설정할까요? 직접 조정하고 싶으면 말해주세요."

승인 → AGENTS.md "## 모델 배치" 섹션에 스킬별 기록
조정 요청 → 사용자가 원하는 배치로 변경
```

## 제약

```
MUST:
  - openclaw.json에서 실제 등록된 모델만 배치 가능 (추측 금지)
  - 배치 변경 전후를 반드시 표시
  - 미등록 모델 배치 요청 시 .env + reload.sh 등록 안내
  - 변경 결과를 memory/에 기록
  - API 키 등록 여부만 표시 ("등록됨"/"미등록"), 키 값은 절대 표시하지 않음

NEVER:
  - 미등록 모델로 배치 설정
  - API 키 값 대화 표시
  - openclaw.json 직접 수정 안내
```
