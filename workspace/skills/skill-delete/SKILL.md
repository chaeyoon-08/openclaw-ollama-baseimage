# 스킬 삭제

> 트리거: "~스킬 삭제해줘", "~기능 없애줘", "~스킬 지워줘"

---

## 시스템 스킬 보호 게이트

```
□ 삭제 대상이 시스템 스킬인가?
  시스템 스킬 목록:
    onboarding, profile-editor, rule-editor, skill-creator, skill-editor,
    skill-toggle, skill-delete, memory-manager, agent-router, status-dashboard

  → Yes: "시스템 스킬은 삭제할 수 없습니다. 비활성화를 원하시면 '스킬명 꺼줘'라고 말해주세요."
         (종료)
  → No: Step 1로 진행
```

---

## 플로우

### Step 1: 대상 스킬 확인

```
명확한 경우: 사용자 요청에서 스킬명 추출 → 스킬 폴더 존재 확인
불명확한 경우:
  - skills/ 폴더에서 사용자 스킬 목록 읽기 (시스템 스킬, _template 제외)
  - 목록 표시 → "어떤 스킬을 삭제할까요?" 선택 요청

스킬 미존재:
  → "해당 스킬이 없습니다." (종료)
```

### Step 2: 스킬 상세 정보 표시

삭제 전에 해당 스킬의 전체 정보를 보여준다.

```
"삭제 대상 스킬 정보:"
""
"| 항목 | 값 |"
"|------|-----|"
"| 이름 | [name] |"
"| 설명 | [description] |"
"| 트리거 | [조건] |"
"| 할당 모델 | [모델명] (배치 있는 경우) |"
"| 폴더 | skills/[스킬명]/ |"
"| 파일 수 | [N]개 (SKILL.md + 보충 파일) |"
```

### Step 3: 삭제 확인 (2단계 확인)

```
1차 확인:
  "이 스킬을 삭제할까요? 삭제하면 복구할 수 없습니다."

사용자 승인 후 2차 확인:
  "정말 삭제합니다. '[스킬명] 삭제'라고 입력해주세요."

2차 확인 텍스트가 일치하지 않으면:
  → "삭제를 취소합니다." (종료)
```

### Step 4: 삭제 실행

```
삭제 순서:
  1. skills/[스킬명]/ 폴더 전체 삭제
  2. AGENTS.md "## 모델 배치" 섹션에서 해당 스킬 배치 규칙 제거
  3. MEMORY.md에서 해당 스킬 관련 인덱스 항목 제거
     (memory/ 파일 자체는 삭제하지 않음 — 기록 보존)

각 단계 실행 후 확인:
  □ 폴더가 실제로 삭제되었는가?
  □ AGENTS.md가 정상 수정되었는가?
  □ MEMORY.md가 정상 수정되었는가?
```

### Step 5: 완료 보고

```
"스킬 삭제 완료."
""
"삭제된 스킬: [스킬명]"
"정리된 항목:"
"- 스킬 폴더: 삭제됨"
"- 모델 배치: [제거됨 / 해당 없음]"
"- 메모리 인덱스: [제거됨 / 해당 없음]"
"- 메모리 상세 기록: 보존됨 (memory/[파일명])"
""
"새 세션에서 반영됩니다."
```

### Step 6: 기록

```
memory/YYYY-MM-DD.md에 삭제 이력 추가:
  - 삭제 일시
  - 삭제된 스킬명 + description
  - 삭제 전 설정 요약 (복구 참고용)
```

---

## 제약

```
MUST:
  - 2단계 확인 (일반 확인 + 스킬명 입력 확인) 필수
  - 삭제 전 스킬 상세 정보 표시
  - 삭제 후 관련 배치/인덱스 정리
  - memory/ 상세 기록은 보존 (삭제 이력 추적용)
  - 삭제 이력을 memory/에 기록

NEVER:
  - 시스템 스킬 삭제
  - 1단계 확인만으로 삭제 실행
  - memory/ 상세 기록 파일 삭제 (인덱스만 정리)
  - 확인 없이 삭제
```
