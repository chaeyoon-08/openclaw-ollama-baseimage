# 스킬 수정

> 트리거: "~시간 바꿔줘", "~설정 변경", "~스킬 수정", "~로직 바꿔줘"

---

## 사전 참조

```
SKILL.md 수정 시 반드시 참조:
  skills/_template/SKILL_TEMPLATE.md
    - 수정 후에도 템플릿 규칙을 준수해야 함
```

---

## 플로우

### Step 1: 대상 스킬 확인

```
명확한 경우: 사용자 요청에서 스킬명 추출 → 해당 스킬 폴더 존재 확인
불명확한 경우:
  - skills/ 폴더에서 사용자 추가 스킬 목록 읽기 (_template, 시스템 스킬 제외)
  - 목록 표시 → "어떤 스킬을 수정할까요?" 선택 요청

스킬 폴더 미존재 시:
  → "해당 스킬이 없습니다. 스킬 목록을 확인할까요?"
```

### Step 2: 현재 설정 표시

해당 스킬의 SKILL.md를 실제로 읽고, AGENTS.md "## 모델 배치" 섹션에서 할당된 모델도 확인한다.

```
표시 형식:
  "[스킬명] 현재 설정:"
  ""
  "| 항목 | 값 |"
  "|------|-----|"
  "| 트리거 | [조건] |"
  "| 입력 | [정보] |"
  "| 출력 | [형식] |"
  "| 실행 모델 | [모델명] (기본/지정) |"
  "| 환경변수 | [목록] (등록됨/미등록) |"
  "| 상태 | 활성/비활성 |"
  ""
  "어떤 부분을 수정할까요?"
```

### Step 3: 변경 사항 정리 + 승인

```
표시 형식:
  "변경 사항:"
  ""
  "| 항목 | 현재 | 변경 후 |"
  "|------|------|---------|"
  "| 트리거 | 매일 09:00 | 매일 07:00 |"
  ""
  "이대로 적용할까요?"
```

### Step 4: 승인 → 파일 수정

```
수정 대상 판단:
  - 설정값(트리거, 입력, 출력 등) → SKILL.md frontmatter 또는 본문 수정
  - 실행 모델 변경 → AGENTS.md "## 모델 배치" 섹션 수정
    (등록된 모델 목록을 보여주고 선택. 미등록 모델이면 .env + reload.sh 안내)
  - 실행 로직 → 스크립트 파일 수정 (변경될 코드를 사전에 보여주고 승인)
  - 환경변수 추가/변경 → AGENTS.md 환경변수 변경 안내 프로토콜 적용

수정 시 SKILL_TEMPLATE.md 품질 체크리스트 확인
```

### Step 5: 변경 후 전체 설정 요약

```
표시 형식:
  "수정 완료."
  ""
  "[스킬명] 변경 후 설정:"
  "| 항목 | 값 |"
  "|------|-----|"
  "| 트리거 | [새 값] |"
  "| ... | ... |"
```

### Step 6: 기록

```
기록 대상:
  - 기존 memory/YYYY-MM-DD-[스킬명].md 가 있으면 → 수정 이력 추가
  - 없으면 → memory/YYYY-MM-DD-[스킬명].md 생성

기록 내용:
  - 수정 일시
  - 변경 항목별 이전값 → 새 값
  - 수정 요청 원문

MEMORY.md 인덱스 업데이트
```

---

## 제약

```
MUST:
  - 변경 전후 값을 반드시 함께 표시 (실제 파일에서 현재값 읽기)
  - 로직(스크립트) 수정 시 변경될 코드를 보여주고 승인 후에만 적용
  - 수정 후에도 SKILL_TEMPLATE.md 규칙 준수 확인
  - 수정 후 SKILL.md 글자수 확인 (15,000자 상한, 초과 시 보충 파일 분리)
  - 변경 결과를 memory/에 기록
  - 환경변수 변경 시 AGENTS.md 프로토콜 적용

NEVER:
  - 스킬 삭제 (이 스킬의 범위 아님. 삭제 요청 시 "스킬 삭제는 직접 폴더를 삭제해야 합니다" 안내)
  - 현재값을 파일에서 읽지 않고 추측
  - 승인 없이 파일 수정
```
