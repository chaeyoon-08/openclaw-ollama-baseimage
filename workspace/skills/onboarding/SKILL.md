# 온보딩

> 트리거: 첫 대화 (USER.md 호칭이 "미설정") / "온보딩" / "처음부터 설정" / "초기 설정"

---

## 사전 게이트

```
□ USER.md 호칭이 이미 설정되어 있는가?
  → Yes: "기존 프로필이 있습니다. 새로 설정할까요, 기존 프로필을 유지할까요?"
    - 새로 설정 → Step 1부터 진행
    - 유지 → "알겠습니다." (온보딩 종료)
  → No: Step 1부터 진행
```

---

## 플로우 (반드시 한 단계씩 진행. 다음 단계로 넘어가기 전 사용자 응답 대기)

### Step 1: 인사 + 자기소개

에이전트가 보낼 메시지:

```
안녕하세요. 업무를 돕는 AI 비서입니다.

할 수 있는 일:
- 프로필/톤 설정
- 새 기능(스킬) 추가/수정/관리
- 행동 규칙 설정
- 기억 관리
- 현재 상태 확인

몇 가지 여쭤볼게요.
```

→ 사용자 응답 대기하지 않고 바로 Step 2의 첫 질문으로 이동

### Step 2: 사용자 프로필 수집

순서대로 질문한다. 사용자가 "건너뛰기", "나중에", "패스" 등을 말하면 해당 항목을 건너뛴다.

```
Q1: "어떻게 불러드릴까요?"
  → 응답을 USER.md 호칭에 기록
  → 응답 없음/건너뛰기 → 호칭 "(미설정)" 유지

Q2: "주로 어떤 일 하세요?"
  → 응답을 USER.md 직업/역할에 기록
  → 응답 없음/건너뛰기 → "(미설정)" 유지

Q3: "편하게 반말로 할까요, 존댓말 유지할까요?"
  → 반말 선택 시:
    - USER.md 톤 → "반말"
    - SOUL.md 언어 규칙 테이블의 기본 톤 → "반말"
    - 이 응답부터 즉시 반말 적용
  → 존댓말 선택 시: 기본값 유지 (변경 없음)
  → 건너뛰기 → 존댓말 유지
```

### Step 3: 시스템 상태 표시

아래 정보를 실제로 읽어서 표시한다 (추측 금지):

```
읽기 대상:
  1. /root/.openclaw/openclaw.json → models.providers 키 → 등록된 모델 이름 + provider 추출
  2. /root/.openclaw/workspace/skills/ → 폴더 목록 → _template 제외 → 활성/비활성 구분

표시 형식:
  "현재 시스템 상태:"
  "- 모델: [모델명] ([provider])"
  "  (추가 모델이 있으면 모두 나열)"
  "- 시스템 스킬: [N]개 (프로필 편집, 규칙 편집, 스킬 관리 등)"
  ""
  "추가 AI 모델은 터미널에서 .env 파일에 API 키를 등록하면 사용할 수 있습니다."
```

### Step 4: 스킬 추가 제안

```
메시지:
  "추가하고 싶은 기능이 있나요?"
  "예: '매일 아침 날씨 알려줘', '이메일 요약 기능 추가해줘'"
  ""
  "지금은 괜찮다면 언제든 말해주세요."

사용자 응답 처리:
  → 기능 추가 원함 → skill-creator 스킬로 연계 (자연스럽게 전환)
  → "괜찮아", "나중에" 등 → Step 5로 이동
```

### Step 5: 완료

```
메시지:
  "설정 완료. 필요할 때 언제든 말해주세요."

기록:
  1. memory/user-profile.md 생성/업데이트
    - 수집된 모든 프로필 정보 (호칭, 직업, 톤 선택, 건너뛴 항목)
    - 온보딩 일시
  2. MEMORY.md 인덱스에 추가
    - "사용자 프로필 설정 완료 → memory/user-profile.md"
```

---

## 제약

```
MUST:
  - 한 단계씩 진행 (Step 1~5를 한 메시지에 보내지 않음)
  - 실제 파일을 읽어서 시스템 상태 표시 (추측 금지)
  - 프로필 수집 결과를 USER.md에 즉시 반영
  - 톤 변경 시 SOUL.md도 함께 수정
  - 온보딩 결과를 memory/user-profile.md에 기록

NEVER:
  - 사용자가 건너뛰겠다는 항목을 강제로 답하게 하기
  - Step 3에서 등록되지 않은 모델을 표시하기
```
