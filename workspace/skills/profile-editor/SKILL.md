# 프로필 편집

> 트리거: 호칭 변경 ("~라고 불러줘"), 톤 변경 ("반말로", "존댓말로"), 프로필 수정 ("직업 바꿔"), 응답 길이 ("짧게 말해")

---

## 수정 대상 매핑

| 요청 유형 | 수정 파일 | 수정 위치 |
|-----------|-----------|-----------|
| 호칭 변경 | USER.md | 기본 정보 > 호칭 |
| 직업/역할 변경 | USER.md | 기본 정보 > 직업/역할 |
| 톤 변경 (반말↔존댓말) | USER.md + SOUL.md | USER.md 선호 > 톤 + SOUL.md 언어 규칙 > 기본 톤 |
| 응답 길이 변경 | USER.md + SOUL.md | USER.md 선호 > 응답 길이 (SOUL.md 응답 길이 기준 테이블 참조) |
| 출처 표기 변경 ("출처 숨겨줘"/"보여줘") | USER.md | USER.md 선호 > 출처 표기 (숨김/간략/상세) |

---

## 플로우

### Step 1: 변경 항목 파악

사용자 요청에서 변경하려는 항목과 새 값을 추출한다.
한 번에 여러 항목을 변경할 수 있다 (예: "반말로 해줘. 이름은 채윤").

```
파악할 항목:
  - 호칭: 새 호칭 값
  - 직업: 새 직업 값
  - 톤: 반말 / 존댓말
  - 응답 길이: 짧게 / 보통 / 자세히
```

### Step 2: 현재값 + 변경값 표시 → 승인

USER.md를 읽어 현재 값을 확인하고, 변경 전후를 테이블로 표시한다.

```
표시 형식:
  "프로필을 변경합니다."
  ""
  "| 항목 | 현재 | 변경 후 |"
  "|------|------|---------|"
  "| 호칭 | (미설정) | 채윤 |"
  "| 톤 | 존댓말 | 반말 |"
  ""
  "적용할까요?"
```

### Step 3: 승인 → 파일 수정

사용자 승인 후:

```
수정 순서:
  1. USER.md 해당 필드 값 변경
  2. 톤 변경인 경우 → SOUL.md 언어 규칙 테이블의 "기본 톤" 행 값 변경
  3. 응답 길이 변경인 경우 → USER.md 선호 설정 > 응답 길이 값 변경
```

### Step 4: 변경 후 전체 프로필 표시

변경된 항목뿐 아니라 전체 프로필을 표시한다.

```
표시 형식:
  "변경 완료."
  ""
  "[현재 프로필]"
  "- 호칭: 채윤"
  "- 직업: (미설정)"
  "- 톤: 반말"
  "- 응답 길이: 보통"
```

### Step 5: 기록

```
기록 대상: memory/user-profile.md
기록 내용:
  - 변경 일시
  - 변경 항목별 이전값 → 새 값
  - 변경 요청 원문 (사용자가 한 말)

MEMORY.md 인덱스:
  - 기존 "사용자 프로필" 항목이 있으면 업데이트
  - 없으면 추가: "사용자 프로필 → memory/user-profile.md"
```

---

## 제약

```
MUST:
  - 변경 전후 값을 반드시 함께 표시 (현재값을 실제 파일에서 읽기)
  - 승인 후 해당 응답부터 즉시 새 톤 적용
  - SOUL.md 수정 시 언어 규칙 / 응답 스타일 섹션만 변경
  - 변경 결과를 memory/user-profile.md에 기록

NEVER:
  - SOUL.md의 행동 경계, 레드라인 섹션 수정
  - 승인 없이 파일 수정
  - 현재값을 확인하지 않고 변경 (파일을 읽지 않고 추측으로 "현재 존댓말입니다" 등)
```
