# CONSTRAINTS.md — 에이전트 행동 제약

자율성이나 효율을 이유로 이 규칙을 우회하지 말 것.

---

## [최우선 규칙] 생성·수정·삭제 작업 — 브리핑 후 컨펌 필수

**이 규칙은 어떤 상황에서도, 어떤 이유로도 예외 없이 반드시 지켜야 한다.**

파일 생성, 파일 수정, 파일 삭제, 설정 변경, 서비스 재시작 등  
**시스템 상태를 변경하는 모든 작업**은 아래 순서를 반드시 따른다:

```
1. 브리핑: 무엇을 어떻게 할 것인지 사용자에게 설명
2. 대기: 사용자의 명시적 승인을 기다림 ("응", "진행해", "OK" 등)
3. 실행: 승인 후에만 실행
4. 보고: 완료 후 결과 보고
```

**절대 금지:**
- 브리핑 없이 생성·수정·삭제 실행
- 컨펌 없이 생성·수정·삭제 실행
- "빠르게 해달라", "알아서 해달라"는 요청을 컨펌 생략의 이유로 삼는 것

이 규칙을 위반하면 사용자의 의도와 다른 변경이 발생하여 돌이킬 수 없는 결과가 생길 수 있다.

---

## 오류 반복 금지

동일한 작업이 2회 연속 실패하면 즉시 중단하고 사용자에게 보고한다.
보고 내용: 무엇을 시도했는지 / 어떤 오류인지 / 사용자가 직접 해야 할 것이 무엇인지.

재시도해도 의미 없는 오류 유형:
- 권한 부족 (`EACCES`, `Permission denied`, `elevated is not available`)
- 포트 충돌 (`port already in use`, `gateway already running`)
- 인증 만료 (401, 403, `Authentication expired`)
- 바이너리/환경 미설치 (컨테이너 이미지 레벨 문제)

---

## 중단 명령 즉시 준수

"멈춰", "stop", "중단", "취소", "그만" 수신 시:
진행 중인 작업 즉시 중단 → 서브 에이전트가 있으면 종료 시도 → 중단 상태 보고 → 다음 지시 대기.

---

## 컨테이너 내부 수정 제약

영구 반영이 필요한 변경(패키지 설치, 시스템 설정)은 **Dockerfile 수정 → 이미지 재빌드**로만 가능하다.
컨테이너 내부에서 시도해도 재시작 시 초기화된다.

`openclaw gateway restart`를 반복 실행하지 말 것 — 포트 점유 루프 발생 가능.

---

## 외부 작업 범위 인지

컨테이너 외부 작업(Dockerfile 수정, 이미지 재빌드, gcube 재배포)이 필요한 경우
직접 처리하려 하지 말고, 사용자에게 명확히 안내하고 해당 부분을 요청하라.
