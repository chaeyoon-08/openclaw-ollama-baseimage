# CONSTRAINTS.md — 에이전트 행동 제약

이 파일은 에이전트가 반드시 준수해야 하는 **하드 제약**이다.
자율성이나 효율을 이유로 이 규칙을 우회하거나 무시하지 말 것.

---

## 오류 반복 금지 (최우선)

**동일한 작업이 2회 연속 실패하면 즉시 중단하고 사용자에게 보고하라.**

```
1회 실패 → 원인 분석 후 다른 방법으로 1회 재시도
2회 연속 실패 → 즉시 중단. 사용자에게 다음을 보고:
  - 무엇을 시도했는지
  - 어떤 오류가 발생했는지
  - 왜 계속 시도하는 것이 의미 없는지
  - 사용자가 직접 해야 하는 것이 무엇인지
```

반복 시도로는 해결되지 않는 오류의 유형:
- 권한 부족 (`elevated is not available`, `EACCES`, `Permission denied`)
- 포트 충돌 (`port already in use`, `gateway already running`)
- 인증 만료 (`Authentication expired`, 401, 403)
- 바이너리/환경 미설치 (컨테이너 이미지 레벨 문제)

---

## 사용자 중단 명령 즉시 준수

사용자가 "멈춰", "중단", "stop", "그만", "취소" 등을 입력하면:

1. 현재 진행 중인 작업을 **즉시** 중단한다
2. 서브 에이전트가 실행 중이라면 종료를 시도한다
3. 중단 상태와 지금까지 완료된 내용을 보고한다
4. 사용자의 다음 지시를 기다린다

모델의 context나 token이 부족하다는 이유로 중단 요청을 무시하지 말 것.

---

## Elevated 권한 도구 실패 처리

`elevated is not available` 오류가 발생하면:

- **재시도 금지.** 권한 구조가 바뀌지 않는 한 재시도해도 결과가 동일하다.
- 즉시 사용자에게 보고: "이 작업은 root 권한이 필요하며 현재 실행 환경에서는 불가능합니다."
- 대안 제시: Dockerfile 수정 → 이미지 재빌드 방식을 권장한다

---

## 컨테이너 내부 시스템 수정 제약

컨테이너 내부에서 다음 작업은 시도하지 말 것:

| 금지 작업 | 이유 |
|---|---|
| `uv tool install`, `pip install` (강제) | 컨테이너 재시작 시 초기화됨. Dockerfile 수정이 올바른 방법 |
| `openclaw gateway restart` (반복) | 이미 실행 중인 gateway와 충돌, 포트 점유 무한 루프 발생 가능 |
| `openclaw.json` 직접 수정 후 gateway 재시작 | config.patch + 자동 reload로 처리 — 수동 재시작 불필요 |
| root 권한 필요 작업 | node 사용자로 실행 중 — 불가능 |

영구 반영이 필요한 패키지 설치나 시스템 설정 변경은 반드시 **Dockerfile 수정 → 이미지 재빌드** 방식을 사용하라.

---

## 작업 복잡도 자가 평가

작업 시작 전 다음을 스스로 평가하라:

- 이 작업이 컨테이너 내부에서 완료 가능한가? (영구성 여부)
- 실패 시 롤백이 가능한가?
- 사용자에게 직접 수행을 요청해야 하는 부분이 있는가?

컨테이너 외부 작업(Dockerfile 수정, 이미지 재빌드, gcube 재배포)이 필요하다면 사용자에게 명확히 안내하고 해당 부분을 직접 처리하도록 요청하라.
